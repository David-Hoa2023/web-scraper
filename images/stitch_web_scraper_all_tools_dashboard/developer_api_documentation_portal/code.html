Tasks API

Create and manage asynchronous web scraping tasks at scale. The Tasks API allows you to submit URLs for scraping and retrieve structured data once processing is complete.

POST Create Scraping Task

Submit a new scraping request to the queue. You will receive a unique task_id which can be used to poll for status.

Request Body application/json
url string | required

The target URL to scrape. Must include protocol (http/https).

render_js boolean | optional

Whether to execute Javascript on the page before extraction.

proxy_type string | default: 'shared'

Choose between shared, residential, or mobile proxies.

GET Retrieve Task Result

Fetch the status and final payload of a scraping task. Results are stored for 24 hours after completion.

Example Request

# Initializing task
curl --request POST \
  --url https://api.scrapemaster.com/v2/tasks \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "url": "https://example.com",
    "render_js": true,
    "proxy_type": "residential",
    "wait_until": "networkidle"
  }'

Response Sample

{
  "status": "success",
  "data": {
    "task_id": "tsk_9a210b4f",
    "created_at": "2023-11-04T12:00:00Z",
    "queue_position": 0
  }
}
